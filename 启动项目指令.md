# DirectMe 可视化 Demo 启动指令

当前仓库已经补了一套最小占位 demo 资产，所以即使没有原始
`directme_scal3r_full_pipeline/` 导出目录，也能先把 3D 页面启动起来。
后续如果拿到真实数据，直接用真实目录替换仓库根目录下的
`directme_scal3r_full_pipeline/` 即可。

前提：

- 使用仓库根目录：`/home/hachimi22/project/scene_graph_gen`
- 本机有 `python3` 即可；启动 `web_server` 不依赖额外 pip 包
- 页面里的 `three.js` 走 CDN，如果浏览器完全离线，前端 3D 脚本不会加载

## 3D 汇报页面

如果需要重新生成与当前 scene graph 坐标一致的 dense map，使用仓库已经配好
EXR 读取依赖的 `.venv` 生成 demo PLY：

```bash
PYTHONPATH=. .venv/bin/python -m directme.demo.export_dense_map
```

它会生成：

```text
directme_scal3r_full_pipeline/dense_pointcloud_world.ply
```

如果这个文件缺失，页面仍会加载 scene graph，但会跳过不匹配的旧 semantic
point cloud，而不是把旧点云叠到新轨迹上。

在仓库根目录运行：

```bash
python3 -m directme.demo.web_server --port 8765
```

打开：

```text
http://127.0.0.1:8765/directme/demo/web/
```

当前汇报页面左侧会按顺序展示三路感知视频视图：Original Video、Depth Map
Video、Object Tracking Video。右侧 Question Timeline 已改为英文展示。

如果你看到的是占位场景图，说明服务已经正常工作，只是当前还没换成真实 SCAL3R 数据。

## 坐标调试页面

同一个服务启动后，打开：

```text
http://127.0.0.1:8765/directme/demo/web/debug.html
```

调试页面可以通过滑条分别调整 ego arrow/trajectory、scene graph 和 dense point cloud 的平移、旋转、缩放。点击 `保存配置` 后，会写入：

```text
directme/demo/web/alignment_config.json
```

刷新 3D 汇报页面后会自动读取这份配置。

## 只验证命令行 demo

这一步使用仓库 `.venv`：

```bash
PYTHONPATH=. .venv/bin/python -m directme.demo --qa-json ''
```

## 4bit Qwen3-VL 4B 推理 demo

仓库内已有 `models/qwen_vl_4B` 时，在仓库根目录运行：

```bash
PYTHONPATH=. python3 -m directme.demo --mode qwen --model-path models/qwen_vl_4B --load-in-4bit --vlm-frame-budget 32
```

当前机器已经补齐该命令需要的 `torchvision` 和 `jinja2>=3.1.0` 用户级依赖。

这个命令会复用现成的
`directme_scal3r_full_pipeline/directme_mapping_run/scene_graph.json`，并把最多
32 张 demo keyframe 送入本地 Qwen3-VL。命令行会打印每条 QA 的耗时和
`QA_FPS`。汇报页面顶部工具栏会实时显示浏览器渲染 `FPS`。
